## OVERVIEW

Success metrics are organized by phase and function. Each metric includes:
- **Definition:** Clear calculation
- **Target:** Aggressive but achievable goal
- **Baseline:** Starting point
- **Cadence:** How often measured
- **Owner:** Who tracks it
- **Red Flag:** When to alert

## 🎯 PHASE 0-1 METRICS (MVP LAUNCH)

## 🎯 NORTH STAR METRIC

### THE ONE METRIC THAT MATTERS

**Primary North Star:** Monthly Active Users (MAU)

**Why:**
1. **Reflects Core Value:** Users return because product is valuable
2. **Predictive:** Strong MAU → future revenue, retention, virality
3. **Actionable:** All features tie back to engagement
4. **Visible:** Easy to track, communicate, celebrate
5. **Balanced:** Prevents optimizing one metric at expense of others

## 🚨 ALERT THRESHOLDS

| Metric | Red Flag | Action |
|--------|----------|--------|
| Crash Rate | >1% | P0 incident, rollback if needed |
| DAU | <25% of MAU | Engagement investigation |
| D7 Retention | <25% | UX audit, onboarding redesign |
| CPA | >₹100 | Channel audit, shift budget |
| Support Tickets | >20 per 1K users | Product investigation |
| GTV | -20% MoM | Investigate churn, product issue |
| App Rating | <3.5 stars | Urgent quality fix required |
| First Activation | >3 days | Onboarding friction analysis |
